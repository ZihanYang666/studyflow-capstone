Working software product

StudyFlow helps track homework before it becomes overwhelming.

This capstone product is a small study planner. It lets a user add school tasks, set priority, mark work complete, filter the list, and see progress update instantly.

Demo move: Add one assignment, set it to high priority, then mark it complete to show the progress bar change.

    Capstone goal

    Build a useful school tool that solves a real organization problem.

    The goal is to design, build, test, and present an interactive homework tracker that helps a student manage tasks by course, due date, priority, and completion status.

    Problem

    Assignments from different classes can be hard to remember and prioritize.

    Product

    A browser-based planner that stores tasks and shows visible progress.

    Impact

    Make schoolwork feel more manageable by turning a messy list into clear next steps.

    Research

    The build used research about web structure, design, interaction, and local storage.

    HTML

    Used for form fields, labels, buttons, sections, and the task list structure.

    MDN HTML

    CSS

    Used for layout, responsive design, visual hierarchy, spacing, and readable states.

    MDN CSS

    JavaScript

    Used to add tasks, filter tasks, save data, update progress, and handle clicks.

    MDN JavaScript

    Success criteria

    The final product can be judged by whether it works for a real user.

    Criteria
    Evidence
    User can add a new task with course, date, and priority.
    The form creates a new card in the task list.
    User can mark a task complete and see progress change.
    The progress bar and completion count update immediately.
    User can filter tasks by status or high priority.
    The filter buttons change which tasks are visible.
    The design is understandable on desktop and mobile.
    The layout uses responsive grids and clear labels.

    Reflection

    Changing the topic made the outcome easier to demonstrate.

    A study planner is easier to present because the audience can immediately see the problem, test the product, and judge whether the solution works.

    The next improvement would be adding categories such as "Today", "This week", and "Overdue", then testing the planner with classmates for feedback.
